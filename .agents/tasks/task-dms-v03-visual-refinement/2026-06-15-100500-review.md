# Rich cabin simulation and visual density upgrade for DMS Cockpit v0.3

The CenterVideoPanel was rewritten from a minimal wireframe-over-div layout into a single full-SVG cabin scene with driver silhouette, detailed interior geometry, facial landmarks, scan-line and noise overlays, and multiple SVG filters. The RightHeadPanel gained a denser wireframe mesh, larger model viewport, animated eye glow, and state-reactive border. LeftConsole introduced section headers (via a CSS pseudo-element pattern), larger state badges, glowing progress bars, and scaled-up behavior icons. BottomTimeline added per-sparkline value readouts and glowing alert markers. A handful of CSS utility classes and keyframes were added to `index.css`.

Watch for: SVG filter performance in the center panel (confirmed — seven filter definitions including noise turbulence render every frame), unused CSS definitions shipped without any consumer (confirmed — `animate-subtle-glow`, `animate-panel-glow`, `glass-panel`, and `eye-pulse` keyframe are defined but never referenced), and scanline animation running at ~6.7 fps visually via a repeating `<animate>` element which can cause GPU compositor thrash on low-end hardware (likely).

## High-level view

The center panel migrated from a layered HTML/SVG approach (absolute-positioned `<div>` elements floating over an SVG cabin sketch) to a single SVG canvas. This consolidates rendering but introduces a large `<defs>` block — 15 gradients, 7 filters, a pattern, and a marker — that the browser must maintain even when off-screen elements are culled. The facial landmark system and gaze arrow now live inside the SVG coordinate space, which ties layout precision to the viewBox rather than responsive percentage positioning.

The right head panel grew from 220×220 to 260×260 CSS pixels and from 4 wireframe lines to 16 (10 latitude + 6 longitude). Per-eye radial gradients with SVG `<animate>` looping create a constant repaint cost even when telemetry values are static.

The left console replaces raw divider `<div>` elements with a `SectionHeader` component backed by CSS `::before`/`::after` gradient lines. Progress bars carry inline `boxShadow` glow, behavior icons scaled up, and the state badge conditionally triggers `animate-pulse-border`.

The bottom timeline's `Sparkline` component gained a value readout (last data point × 100, displayed as percentage) and a new `isFirst` prop for border suppression. Alert markers widened to pill shape with colored box-shadow glow.

## CenterVideoPanel: SVG filter budget and rendering cost

The panel's `<defs>` block declares `noiseFilter` (uses `feTurbulence` with `baseFrequency="0.85"` and `numOctaves="4"`), `shadowFilter`, `softGlow`, `cyanGlow`, `greenGlow`, and `cyanDotGlow`. The noise texture is applied via a full-viewport `<rect>` at the end of the SVG, meaning every repaint of the panel triggers turbulence generation. On an embedded automotive display or tablet HMI (the likely target for a DMS cockpit), this can consume 10-20ms per frame depending on GPU. The filter could be replaced with a static pre-rendered PNG noise overlay or a CSS `background-image` repeated tile.

The scanline overlay uses:

```xml
<rect x="0" y="0" width="800" height="500" fill="url(#scanlines)" opacity="0.6">
  <animate attributeName="y" from="0" to="4" dur="0.15s" repeatCount="indefinite" />
</rect>
```

A 0.15s duration shifting only 4 SVG units produces a jittery sub-pixel movement rather than a cinematic scanline sweep. The existing CSS `animate-scanline` keyframe (defined in the base code, 4s duration, full translateY) achieves the effect more smoothly. The SVG `<animate>` element also keeps the SVG renderer's animation loop running regardless of React lifecycle or visibility.

## SVG id collision across component instances

All gradient and filter definitions use static string ids: `bgGrad`, `windshieldGrad`, `cyanGlow`, `gazeArrowHead`, etc. In a React SPA where the component tree might render multiple panels (e.g., a multi-camera DMS setup, or hot-module-reload duplicating the DOM), these ids collide. The browser resolves `url(#bgGrad)` to the first matching element in document order, silently breaking fills on subsequent instances. A common fix is to scope ids with `useId()` (React 18+) or a unique prefix.

The same issue applies in `RightHeadPanel` which defines `leftEyeGlow`, `rightEyeGlow`, and `gazeConeGradient` without namespacing.

## Unused CSS shipped in index.css

- `animate-subtle-glow` — no element applies it.
- `animate-panel-glow` — `RightHeadPanel` uses inline `style={{ boxShadow: ... }}` instead of this class.
- `glass-panel` — unused anywhere.
- `@keyframes eye-pulse` — the eye animation in RightHeadPanel is done via inline SVG `<animate>` attributes, not this CSS keyframe.

The implementation drifted from the original plan without cleaning up the CSS.

## Dead `level` field in getStateOverlayLabel

The function signature was changed from returning a plain string to returning `{ label: string; level: string }`, but the component destructures only `label`:

```tsx
const { label: stateLabel } = getStateOverlayLabel(data.driverState);
```

The `level` field (values: `'normal'`, `'monitor'`, `'warning'`, `'danger'`) is presumably meant to drive the state badge color in the overlay, but the conditional styling below still switches on `stateLabel === 'Normal'` string comparisons. This is a partial refactor left incomplete — either use `level` to drive the styling or revert to the flat string return.
